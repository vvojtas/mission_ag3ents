# Playground

A scratch area for verifying that `common/` components work correctly, experimenting with OpenRouter model responses, and testing new ideas before integrating them into tasks.

## Usage

```powershell
uv run python -m playground.solution
```

## What to use it for

- Smoke-testing `LLMClient`, `HubClient`, `ToolsLoop`, etc.
- Comparing model outputs / costs for a quick prompt.
- Trying out new tools or prompt templates before wiring them into a real task.
- Checking raw OpenRouter API responses and token/cost metadata.
